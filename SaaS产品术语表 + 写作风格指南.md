# SaaS产品术语表 + 写作风格指南 #
## 项目说明

- **适用产品**：假设为一款名为 "WorkFlow Pro" 的SaaS项目管理工具

- **目标读者**：产品文档编写者、市场文案、客服人员

- **项目目标**：统一团队内术语使用和写作风格，确保所有对外内容保持一致

## 第一部分：SaaS 中英术语对照表
以下术语适用于 WorkFlow Pro 产品的所有对外文档、界面文案和市场材料。

| :--- | :--- | :--- | :--- |

## 第二部分：SaaS 中文写作风格规范

### 1. 称呼

| 规则 | 说明 | 正确示例 | 错误示例 |
| :--- | :--- | :--- | :--- |
| 统一用 **"你"** | 不要用"用户"指代当前操作者 | "你可以创建多个项目" | "用户可以创建多个项目" |
| 需要指代用户群体时用 **"用户"** | 讨论用户行为分析等场景 | "新用户注册后需完成新手引导" | "新玩家注册后需完成新手引导" |
| 提及公司时用 **"我们"** | 区分"你"（用户）和"我们"（服务提供方） | "我们会在 24 小时内处理你的请求" | "我会在 24 小时内处理" |

### 2. 语态

| 规则 | 说明 | 正确示例 | 错误示例 |
| :--- | :--- | :--- | :--- |
| 用 **主动语态** | 谁做了什么，不要用"被"字句 | "管理员可以邀请成员" | "成员可以被管理员邀请" |
| 用 **"可以"而不是"能够"** | 保持口语化自然 | "你可以使用看板视图" | "你能够使用看板视图" |
| 指令用 **祈使句** | 隐含未说出的"你" | "单击确定以保存配置" | "单击确定时，配置将被保存" |

### 3. 日期与数字

| 规则 | 示例 |
| :--- | :--- |
| 日期格式统一为 **YYYY-MM-DD** | 2026-06-24 |
| 时间用 **24小时制** | 14:30（不要写下午2:30） |
| 数字统一用 **阿拉伯数字** | 5个成员（不要写"五个成员"） |
| 超过4位的数字加**千位分隔符** | 10,000（不要写10000） |

### 4. 标点符号

| 规则 | 正确示例 | 错误示例 |
| :--- | :--- | :--- |
| 中文环境用**全角标点** | "创建项目后，你可以添加任务。" | "创建项目后,你可以添加任务。" |
| 句号用 **。** 不用 **.** | "已完成。" | "已完成." |
| 列表项末尾**不加句号** | "- 创建项目" | "- 创建项目。" |
| 引用或强调用 **「 」** | 请在「设置」中完成配置 | 请在"设置"中完成配置 |
| 中英文之间**加一个空格** | "你可以在 仪表盘 查看进度" | "你可以在仪表盘查看进度" |
| 数字与单位之间**加一个空格** | "10,000 个项目" | "10000个项目" |

### 5. 句子与段落

| 规则 | 说明 | 示例 |
| :--- | :--- | :--- |
| 句子长度**不超过25个字** | 过长句子拆分为短句 | 错误："本产品适用于从由一台服务器进行动作控制的单一节点结构到由多台服务器进行动作控制的并行处理程序结构等多种体系结构。" |
|  |  | 正确："本产品适用于多种体系结构。无论是由一台服务器，还是由多台服务器进行控制，均可以使用本产品。" |
| 段落**不超过3句话** | 过长的段落要拆分 | 一个大段拆成2-3个小段 |
| 多用**项目符号列表** | 适合列举步骤或要点 | 操作步骤用有序列表，并列信息用无序列表 |
| 用**肯定句**代替否定句 | 直接告诉用户该做什么 | 错误："请确认没有接通电源。" |
|  |  | 正确："请确认电源已关闭。" |
| 避免**双重否定句** | 会让读者困惑 | 错误："没有删除权限的用户不能删除此文件。" |
|  |  | 正确："用户必须拥有删除权限，才能删除此文件。" |

### 6. 禁用词列表

以下词汇在正式文档中禁止使用：

| 禁用词 | 原因 | 替代方案 |
| :--- | :--- | :--- |
| 稍微、大概、差不多 | 技术文档需要精确 | 给出具体数值或范围 |
| 很快、稍后 | 模糊的时间承诺 | 说明具体时间，如"将在24小时内处理" |
| 非常、极其 | 主观修饰词，无信息量 | 直接描述事实 |
| 可能、也许 | 削弱可信度 | 确认后给出确定表述 |
| 大家、小伙伴们 | 过于口语化，不专业 | 用"你"或"用户" |
| 搞、整、弄 | 口语化动词 | 用"创建""完成""进行"等正式动词 |
| 酷毙了、给力 | 非正式语言风格 | 使用中性、专业的表述 |
| 利用 | 可以用更简单的词 | 用"使用"替代"利用" |

### 7. 格式规范

| 规则 | 示例 |
| :--- | :--- |
| 按钮名称用 **加粗** | 点击 **创建项目** 按钮 |
| 界面路径用 **>** 连接 | 进入 设置 > 成员管理 > 添加成员 |
| 代码/命令用反引号包起来 | 在终端输入 `npm install` |
| 提示信息用 `>` 引用块 | > 提示：首次创建项目需要填写项目名称。 |
| 操作步骤用有序列表 | 1. 打开设置 2. 点击成员管理 |
| 术语首次出现时**括号注明英文** | "软件即服务（Software as a Service，简称 SaaS）" |


## 第三部分：使用示例

### 应用前（不符合规范的写法）

> 大家好，欢迎使用 WorkFlow Pro！这是一个很牛的项目管理工具，你可以创建项目，添加任务，还能看甘特图，非常方便。大概5分钟就能上手。大家赶紧邀请小伙伴加入工作区一起协作哦！

### 应用后（符合规范的写法）

> 欢迎使用 WorkFlow Pro！
>
> 你可以在 ***仪表盘*** 创建项目，并为每个项目添加任务。项目进展可通过 ***看板*** 或 ***甘特图*** 直观查看。
>
> 如需协作，你可以在 ***设置 > 成员管理*** 中邀请成员加入当前工作区。
>
> 提示：首次使用建议先完成新手引导，约需 5 分钟。
